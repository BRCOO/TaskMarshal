# Worker Self-Review

Run this checklist before returning work to Codex.

- Scope: Did I touch only files allowed by the task spec?
- Secrets: Did I avoid printing, copying, or modifying secrets?
- Architecture: Did I avoid making decisions outside the assigned task?
- Minimality: Is the change smaller than a broad refactor?
- Tests: Did I run the requested checks or explain why they could not run?
- Diff: Can Codex review the changed files quickly from my summary?
- Blockers: Did I clearly name anything that still needs Codex or user input?
